LAUNCHING THE APPLICATION

Option 1: Visual Studio
  1. Open "Web Browser\Web Browser.sln"
  2. Press F5 to start debugging
  3. Application window appears

Option 2: Command Line
  1. Open PowerShell/Command Prompt
  2. Navigate to project directory
  3. Type: dotnet run
  4. Application launches

Option 3: Direct Execution
  1. Navigate to bin\Release\net472 folder
  2. Double-click "Web Browser.exe"
  3. Application starts immediately

USING THE DOWNLOAD MANAGER

1. Navigate to any website
2. Click on a file link (image, PDF, etc.)
3. Download Manager window appears
4. Watch real-time progress
5. Use controls to manage downloads:
   - Open Folder: Show downloads directory
   - Pause: Temporarily stop download
   - Resume: Continue paused download
   - Cancel: Stop and cleanup
   - Clear Completed: Remove finished items

                         KNOWN LIMITATIONS
???????????????????????????????????????????????????????????????????????????????

CURRENT LIMITATIONS
1. Uses Internet Explorer engine (some modern sites may not render)
2. Limited to 3 concurrent downloads
3. No bandwidth throttling
4. No download resume from partial
5. No download scheduling
6. No proxy support

WORKAROUNDS
1. For modern sites: Open in external browser option available
2. For more downloads: Sequential downloads work (slower but functional)
3. For throttling: Manually limit downloads to 1-2
4. For resume: Delete partial and restart
5. For scheduling: Plan downloads within session
6. For proxy: Use system proxy settings
